Executando bibliotecas nativas com JNA(Java Native API)

27/01/2017
#Java #JNA #JavaNativeAPI #NativeLibrary #dll

Este é um exemplo básico de como fazer chamadas a biblioteca nativas do sistema operacional, como DLL do windows, por exemplo. Interagir com aplicações desenvolvida em outras linguagens a partir do código Java utilizando JNA(Java Native API).

Embora sabemos que o Java é uma linguagem projetada para ser independente da plataforma, fazer chamada a biblioteca nativas do sistema operacional é muito útil quando você precisa interagir bibliotecas específicos do sistema operacional ou quando você precisa interagir com aplicações desenvolvida em outras linguagens, um sistema legado desenvolvido em C, por exemplo. É claro, que integração com um sistema legado pode ser feita de N formas, como arquivo texto, banco de dados e web services, por exemplos. Mas, suponhamos que você queira apenas executar um método específico de uma biblioteca. Um exemplo bastante aplicável disso seria a comunicação com hardware, onde o fabricante, ao invés de disponibilizar um biblioteca para cada linguagens, disponibiliza apenas uma biblioteca nativa que prover a comunicação. Neste caso, para interagir com o hardware seria necessário executar chamadas aos métodos da biblioteca disponibilizada pelo o fabricante.

O Java provê duas forma de executar chamadas a bibliotecas nativas: via JNI (Java Native Interface) e JNA (Java Native API). Neste exemplo utilizaremos a JNA por ser menos complexa. Na verdade, a JNA abstrai a complexidade da JNI tornando o código mais simples e independente do Java, mas no frigir dos ovos o resultado é o mesmo.

Para este exemplo, utilizaremos uma dll já pronta chamada library.dll que pode ser baixada aqui. Esta dll foi desenvolvido em utilizando a linguagem C. Não entrarei em detalhes de como gerar um dll, pois, este seria um assunto para um post futuro.

Então… vamos ao que interessa! Adicione a lib do JNA ao seu projeto. Podemos obtê-la através do repositório oficial do JNA. Cópie a biblioteca para o classpath do seu projeto. Se estiver utilizando o sistema operacional windows, uma alternativa sería colocar a biblioteca no diretório system32.

Fazer uma chamada a uma biblioteca nativa com JNA é muito simples. Primeiramente, precisamos criar uma interface que herda de “com.sun.jna.Library” e definir os métodos com a mesma assinatura definida na biblioteca. O segundo passo é carregar a biblioteca, podemos fazer isso da seguinte forma:

NativeLibrary library = (NativeLibrary) Native.loadLibrary(“library”, NativeLibrary.class);

Como podemos perceber, o método loadLibrary recebe dois parâmetros uma String e uma Class. O primeiro é o nome da biblioteca que desejamos carregar. Um detalhe importante é que devemos passar apenas o nome simples da biblioteca sem a extensão. O segundo parâmetro é a interface que representa biblioteca, ou seja, que contém os métodos com as mesma assinaturas definida na biblioteca. No nosso caso, a interface NativeLibrary.
